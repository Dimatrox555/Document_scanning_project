Объяснения по работе каждого модуля

1.      OpenCV_proj

Описание:
Данный модуль занимается чтением изображения, а затем приведению его в читаемый для модуля ocrnlp вид.
Выполняется это за счёт:
1) Обнаружения углов изображения при помощи нейронной сети
2) Трансформации изображения по углам (коррекция перспективы, вращение, обрезка краёв)
3) Бинаризации изображения (близкие к чёрному цвета становятся чёрным пикселями, близкие к белому - белыми)

Описание переменных:
1) points
Переменная, в которой хранятся координаты углов изображения
2) destination_points
Константа, которая хранит фиксированные координаты, в которые углы изображения будут трансформироваться.

Описание функций:
1) read(img_path, json_path, k)
* img_path - название изображения
* json_path - название разметки
* k - масштабирование изображения в k раз
Данная функция принимает в себя изображение любого формата, а также json-файл разметки
Изображение переводится в матрицу точек - основной тип данных в OpenCV, после чего изображение масштабируется и запоминает h и w - высоту и ширину изображения.
После этого, из разметки достаётся необходимый элемент массива наборов точек и сохраняется в переменной points.
Разметка LabelStudio сохраняет координаты в процентах, из-за чего координаты в points делятся на (100 * w) или (100 * h) в зависимости от того, x или y координата считается.
Все наборы координат сохраняются в NumPy массиве, т.к. это нужно для трансформации изображения.
В итоге, мы обновляем переменную points и получаем на выводе img_output, т.е. матрицу пикселей изображения.

2) image_transform(target, k)
* target - изображение
* k - масштабировать расширение изображения в k раз
Данная функция трансформирует изображение и углы из points перемещает в места, назначенные в destination_points.
Сначала создаётся матрица перспективы из углов и точек назначения
Затем, по этой матрице само изображение трансформируется.
На выходе функция даёт скорректированную перспективу, вращение и обрезку углов на новом изображении

3) grayscale(image)
* image - изображение
Данная функция бинаризует изображение.
Сначала, цвета на изображении конвертируются в чёрно-белый цвет, затем при помощи threshold яркие пиксели становятся белыми, а тёмные - чёрными.
На выходе функция выдаёт бинаризированное изображение.

4) preprocess(img_path, k)
* img_path - путь к изображению (его название, на самом деле)
* k - масштабировать изображение в k раз
Данная функция предназначена для вызова из других модулей, т.к. она совмещает в себе весь необходимый функционал
Сначала, функция определяет все абсолютные пути для папки, откуда берётся разметка и изображения и куда будет сохраняться предобработанное изображение.
Затем, изображение обрабатывается всеми выше перечисленными функциями
В конце, изображение сохраняется в папке processed_images в формате .tiff, а функция возвращает путь к этому же изображению.


Что планируется добавить:
1) Обработка массива изображений
2) ИИ-обнаружение углов фотографии
3) Полная независимость от костыльных аргументов в preprocess()



2.      ocrnlp

Описание:
Данный модуль занимается распознаванием текста с предобработанного изображения и формированием структурированного результата в формате JSON.
Выполняется это за счёт:

Распознавания текста на изображении средствами Tesseract OCR
Нормализации распознанного текста — исправление артефактов сканирования, замена визуально схожих символов
Извлечения полей документа по якорным фразам через регулярные выражения
Формирования итогового JSON документа с извлечёнными данными

Описание переменных:

patterns
Словарь, содержащий библиотеку паттернов и схемы их применения для каждого типа документа
replacements
Словарь замен визуально схожих латинских символов на кириллические аналоги

Описание функций:

ocr_with_confidence(img)


img — изображение в формате PIL или numpy array
Функция распознаёт текст на изображении и возвращает среднее значение уверенности распознавания и распознанный текст


normalize_text(text)


text — распознанный текст
Функция применяет словарь замен и дополнительные правила нормализации к тексту


get_pattern(doc_type, field, pattern_index)


doc_type — тип документа
field — название поля
pattern_index — индекс паттерна в списке
Функция извлекает нужный паттерн из библиотеки по схеме типа документа


extract_document(text, doc_type, field, pattern_index)


text — нормализованный текст
doc_type — тип документа
field — название поля
pattern_index — индекс паттерна
Универсальная функция извлечения поля из текста по якорному паттерну. Заменяет индивидуальные геттеры


build_json(text, save_file)


text — распознанный текст
save_file — флаг сохранения результата в файл
Функция формирует итоговый JSON документ из всех извлечённых полей


process_image(img)


img — путь к изображению
Основная функция модуля для вызова из других модулей. Запускает весь пайплайн от изображения до JSON
